scan_jobs · scan_modules · scan_job_modules (pivot)
0/0
▼
🐳
How Scan Execution Works
User calls API → Controller creates scan_job → dispatches RunScanJob to Redis queue → Queue Worker picks it up → for each module: runs docker run cyberguard/{module} {target} → script inside container runs the tool (nmap / subfinder / etc.) → outputs JSON to stdout → Laravel reads stdout → creates Finding records → broadcasts ScanCompleted via WebSocket → frontend updates live.
⚙️
Infrastructure Setup
ONE-TIME SETUP
✓
Set QUEUE_CONNECTION=redis in .env — install predis/predis package
required
✓
Set BROADCAST_DRIVER=pusher (or use laravel-websockets for self-hosted) — install pusher/pusher-php-server
required
✓
Make sure Docker is installed on the server and the Laravel process user can run docker run — add user to docker group
required
✓
Create a dedicated Docker network: docker network create cyberguard-scan-net --driver bridge --internal
--internal means containers on this network have NO internet access — scan target only via explicit routing
security
✓
Seed scan_modules table — one row per module with slug, name, docker_image, category, is_active = true
slugs: sqli, xss, csrf, port-scan, ssl-check, subdomain-enum, http-headers, cors-check, banner-grab, whois, dir-bruteforce, cve-lookup
required
✓
Run queue worker with Supervisor: php artisan queue:work redis --queue=scans --timeout=300 --tries=1
required
