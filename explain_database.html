Core / Auth / Multi-Tenant
المستخدمين، الشركات، والاشتراكات
01
users
بيانات كل شخص بيستخدم المنصة
CORE ▼
تخيله زي: بطاقة هوية — فيها كل معلومات الشخص، إيميله، كلمة السر، وآخر مرة دخل.
اسم العمودالنوعالشرح
🔑 id UUID معرف فريد لكل مستخدم — مثل رقم قومي مش بيتكرر
email VARCHAR UNIQUE الإيميل — لازم يكون فريد، مينفعش اتنين بنفس الإيميل
password VARCHAR كلمة السر — بتتحفظ مشفرة (Hash) مش نص عادي
name VARCHAR اسم المستخدم اللي بيظهر في الـ Dashboard
avatar_url TEXT رابط صورة البروفايل
email_verified_at TIMESTAMP إمتى اتأكد من الإيميل — لو NULL معناها لسه ما اتأكدش
two_factor_secret VARCHAR الكود السري لميزة الـ 2FA (التحقق بخطوتين)
two_factor_enabled BOOLEAN هل الـ 2FA شغال؟ true أو false
last_login_at TIMESTAMP آخر مرة المستخدم دخل — بيتحدث كل Login
created_at / updated_at TIMESTAMP Laravel بيعملهم تلقائياً — إمتى اتنشئ وإمتى اتعدل
deleted_at TIMESTAMP Soft Delete — لو فيه قيمة يبقى محذوف بدون ما يتمسح فعلاً
علاقاته
له subscription واحدة عضو في organizations كتير عنده projects شخصية مشارك في projects ناس تانية
مثال: Ahmed بيسجل بـ ahmed@gmail.com — بيتنشئ له سطر في الجدول ده بـ UUID فريد وكلمة السر مشفرة.

start_date / end_dateDATEتاريخ بداية ونهاية المشروع
مثال — مشروع شخصي: owner_type = User, owner_id = Ahmed's UUID
مثال — مشروع للشركة: owner_type = Organization, owner_id = SecureTeam's UUID

10
scan_modules
أدوات الفحص المتاحة في المنصة
SCAN ▼
تخيله زي: قائمة الأدوات في صندوق العدة — SQLi, XSS, Port Scanner... كل أداة ليها اسم وتصنيف.
اسم العمودالنوعالشرح
🔑idUUID
nameVARCHARالاسم الكامل: "SQL Injection Scanner"
slugVARCHAR UNIQUEالاسم المختصر: sqli — بيتستخدم في الكود
categoryENUMweb لثغرات المواقع / network للشبكات / recon للاستطلاع
descriptionTEXTشرح بسيط للأداة
is_activeBOOLEANهل الأداة متاحة حالياً؟ لو false مش هتظهر للمستخدم
أمثلة على سطور في الجدول:
sqli → SQL Injection → web
xss → Cross-Site Scripting → web
port-scan → Port Scanner → network
subdomain → Subdomain Enum → recon
